前端後端的整合潮流

在 PloneConf 2018 活動裡，討論主題之一是 Plone 系統的演化走向，相關議題則是展示 JavaScript 各式框架怎樣跟內容管理系統 REST API 進行整合。

Plone 以 Python 語言寫成，歷經十餘年的發展，是個成熟穩定的內容管理系統，但另一個角度來看，它的架構設計顯得老舊，面臨新時代的轉型挑戰，怎樣與時俱進地整合新技術，例如熱門的 RESTful API 架構，和前端技術 React, Angular 該怎樣分工合作，這些都是倍受關注的議題。在 Reinventing Plone: Roadmap to the Modern Web 講題裡，Timo Stollenwerk 提及 Plone 可能的演化方向：

Guillontina 是一個後端系統，能滿足非同步高效能網頁需求，同時支援 JavaScript 應用程式整合開發。效能是基於 AsyncIO 的底層設計，特別適合 WebSocket 或 MicroService 的應用場景，權限機制也能細緻管控系統流程。
